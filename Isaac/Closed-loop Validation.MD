# Closed-loop Validation

「閉環數位分生（Closed-loop Digital Twin）」的架構願景：不只是從設計到模擬，而是從模擬/實際運行產生的數據，反向修正（Inverse Mapping）物理機構與控制演算法。
- **數據驅動**：實現機構物理參數的逆向優化（Co-design）。

> [!IMPORTANT]
> **數位分生 (Digital Twin) 與傳統模擬的本質差異**：
> 傳統模擬通常是「開環」的單一驗證，而數位分生是**「全物理特性整合」**的閉環驗證，強調虛實世界的即時或循環反饋。

---

## 1. 核心開發框架與 GitHub 專案

- **[Isaac Lab](https://github.com/isaac-sim/IsaacLab)** (原名 Orbit / 基於 Isaac Sim)：這才是你解決物理機構問題的戰場。它擁有強大的物理引擎（PhysX），可以進行「系統辨識」與「強化學習（RL）」。
- **關鍵專案推薦**：[Aerial Gym](https://github.com/ntnu-arl/aerial_gym) & [Pegasus](https://github.com/PegasusSimulator/PegasusSimulator)
- **PyBullet 推薦？**：適合輕量級驗證，但精度與平行模擬能力（如需解決機構共振）建議直上 Isaac Lab。
- **NOTE (Isaac ROS)**：主要用於感測器處理與導航（例如 VSLAM、避障）。它不太能解決「機構共振」或「物理參數優化」，因為它處理的是軟體通訊與算法加速。

這些專案能幫助你實現從「數據」回到「物理參數」的逆向過程。


### 補充說明：Aerial Gym vs. Pegasus Simulator

#### 1. Aerial Gym (由 NTNU ARL 開發)
- **核心定位**：為大規模強化學習 (RL) 而生的平行模擬框架。
- **優點**：
  - **極致的速度**：針對 GPU 平行運算優化，可以同時模擬數千台無人機。如果您需要進行「隨機重心偏移」的壓力測試（例如：模擬 1000 種不同的重心情況），它是首選。
  - **模組化設計**：非常適合快速替換不同的機體結構（URDF），並立即測試其飛行穩定性。
  - **專注於運動策略**：如果您關心的是「機體結構變更後，飛控演算法如何適應」，Aerial Gym 提供的 RL 訓練環境非常完善。
- **缺點**：
  - **動力學簡化**：為了實現極速模擬，它對部分氣動力學細節進行了簡化（通常使用 simplified dynamics）。對於需要「高精度物理回饋」的結構設計（如：機體在劇烈震動下的結構應力表現），其準確度可能不如純物理模擬器。
- **適合場景**：快速迭代與驗證控制器的「魯棒性（Robustness）」，例如「重心稍微偏掉，飛機還能飛嗎？」這類問題。

#### 2. Pegasus Simulator (由 ETH Zurich 團隊相關背景開發)
- **核心定位**：基於 Isaac Sim 的高保真、多功能無人機模擬平台。
- **優點**：
  - **高保真度 (High Fidelity)**：Pegasus 強調對真實物理的還原，包括電池模型、動力系統、感測器模擬等。它與 PX4 等真實飛控系統的整合度極高。
  - **感測器與環境豐富**：如果您的機體設計涉及「鏡頭、光達 (LiDAR) 安裝位置對重心的影響」，Pegasus 提供了更完整的感知模型。
  - **工業標準化**：它更接近真實的硬體在環 (HITL/SITL) 測試需求，適合驗證「硬體規格是否符合設計目標」。
- **缺點**：
  - **訓練速度稍慢**：雖然基於 Isaac Sim，但因為追求細節與物理真實度，其平行化規模不如 Aerial Gym 那樣極致。
- **適合場景**：詳細驗證單一機體的設計合理性，特別是涉及多種感測器佈局、電力消耗，以及對 PX4 等商用/開源飛控韌體的適應性。

#### 比較總結：哪一個更適合您的專案？

| 比較維度 | Aerial Gym | Pegasus Simulator |
| :--- | :--- | :--- |
| **主要功能** | 大規模 RL 訓練、快速控制策略驗證 | 高精度系統模擬、飛控開發與落地 |
| **重心驗證重點** | 驗證重心偏移的「邊界條件」與抗性 | 驗證重心位置對「飛控參數與電池壽命」的實際影響 |
| **技術棧** | 專注於 Isaac Gym GPU 邏輯 | 整合 Isaac Sim 強大的視覺與物理環境 |
| **適合開發階段** | DVT 設計初期的多參數方案篩選 | DVT 設計後期與正式驗證前夕的準確性檢查 |

#### 💡 給您的具體決策建議：

1. **如果您要解決的是「設計迭代」的問題（如：為了修正重心，要在哪裡加裝配重？）：**
   - **建議使用 Pegasus Simulator**。因為它能精確模擬飛機在實際重心下的飛行電耗與姿態表現，適合做結構的「最終校正」。

2. **如果您要解決的是「飛行風險」的問題（如：機體結構變更後的重心變動，會不會造成飛行不穩？）：**
   - **建議使用 Aerial Gym**。利用它的平行模擬特性，設定一組「重心偏移變量」，快速跑出一個性能分佈圖，這能幫助您明確定義出機體的「重心容許區間 (CG Envelope)」。

#### 🚀 最佳實踐方案 (Best Practice)：
既然您的目標是「機體設計驗證」，建議**兩者結合使用**：
1. 先用 **Aerial Gym** 快速過濾設計方案，淘汰那些對重心敏感度太高、導致極端環境下會墜機的設計。
2. 將剩下來的 1~2 個最佳方案匯入 **Pegasus**，進行高精度的物理模擬，確認各項感測器與電池續航參數符合 DVT (設計驗證測試) 需求，再嘗試開模以進行實機打造。

> **研讀提醒**：
> 這兩個專案都在 GitHub 上有完善的文件。研讀原始碼或文獻時，請特別查看它們的 **"Sensor Modeling"** 與 **"Dynamics Configuration"** 部分，這是影響您重心驗證精確度的關鍵因素。

## 2. 關鍵論文參考 (理論基石)

如果你想實作「從數據反向修正機構」，請參考這幾篇論文的路徑：

1. **《Aerial Gym: A Framework for Highly Parallelized Simulation of Aerial Robots》 (2025)**:
   - 解釋了如何在 Isaac 環境中建立**可重構（Reconfigurable）**的無人機平台。
2. **《NeuroBAM: Neural Body Architecture Model》**:
   - 探討如何將「身體結構」與「神經控制」一起優化（Co-design）。
3. **《Sim-to-Real Transfer for Mobile Robots with Reinforcement Learning》**:
   - 探討如何將 Isaac Sim 訓練的結果（針對不均勻動力修正後的參數）遷移到真實機器人上。

## 3. 如何透過 Isaac 解決「共振」？

共振通常源於高頻振動與結構剛性的耦合。在 Isaac Lab 中，你可以這樣做：

- **高頻採集**：將模擬頻率調高（例如 $1\text{kHz}+$），並在馬達節點加入高頻擾動。
- **結構修正（URDF 逆向）**：
  - 在 Isaac 中運行無人機，收集 IMU 的震動頻譜。
  - 利用 **CMA-ES** (共變異矩陣適應策略) 或 **RL 算法**，自動調整 URDF 中的質量分佈（Mass Distribution）與轉動慣量（Inertia）。
  - **目標**：讓結構頻率避開馬達的工作頻率。

## 💡 實踐活動：情境模擬

假設你發現無人機在 $150\text{Hz}$ 時會發生劇烈共振，導致 IMU 數據失效（演算失效）。在 Isaac Lab 中，你會選擇哪種方式來「找到」解決方案？

- **(A) 手動嘗試**：在程式碼中手動嘗試調整機臂長度，每次改完都跑一次模擬。
- **(B) 大規模平行優化**：設定一個範圍（例如機臂長 $20\text{cm} \sim 25\text{cm}$），讓 Isaac 同時跑 1000 種不同的機型，並自動回傳震動最小的那一組。
- **(C) 降低轉速**：把馬達轉速調低，避開共振點。
