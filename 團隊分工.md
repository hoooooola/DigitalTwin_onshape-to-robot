# 數位孿生無人機開發 - 團隊分工與職責

| 負責團隊 / 角色 | 使用工具/技術棧 | 在 DVT (設計驗證與測試) 中的貢獻 |
| :--- | :--- | :--- |
| **硬體機構 (ME)** | Onshape, SolidWorks, ANSYS | 提供精確的機體 CAD 設計與物理參數，確保結構剛度，避免產生致命共振。 |
| **數位孿生/模擬環境工程師 (SimEng)** | onshape-to-robot, URDF/SDF, Isaac Sim, URDF 構建 | **(核心橋樑)** 承接 ME 模型轉換為系統可讀的 URDF。定義關節與物理碰撞網格，搭建 Pegasus/Gazebo 等模擬環境供後端測試。 |
| **飛控/控制工程師 (C&G)** | PX4 (SITL/HITL), Pegasus Simulator, MATLAB | 針對不同的機體設計（如變換軸距、修改動力），在模擬器中快速調整 PID 參數，驗證控制演算法的動態穩定性。 |
| **感知與導航組 (Perception / AI)** | ROS 2, Aerial Gym, OpenCV, Isaac ROS | 利用構建好的虛擬環境（包含光達、相機），驗證原有的避障、SLAM 或路徑規劃算法在機體變動時的「魯棒性」(Robustness)。 |
| **測試與系統工程師 (QA / System)** | 實機測試, ULog 分析 (Flight Review) | 實際試飛並分析 Log。比對實測數據與 Pegasus 模擬數據的差異，找出 **Sim-to-Real Gap**，反饋給各部門修正。 |
